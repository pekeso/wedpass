# WedPass Beta Wedding Checklist

## 1. Document Purpose

This checklist is used before, during, and after every **WedPass V1.0 beta wedding**.

It is designed for practical event readiness.

The checklist helps ensure:

- guest data is ready
- QR codes are prepared
- Event Mode is enabled correctly
- staff devices are ready for offline check-in
- staff understand the process
- media upload is activated
- all check-ins are synced after the event
- feedback is collected

Because WedPass targets Central, East, and West Africa, the beta should support both **English and French** for key operational flows.

# Part 1 — One Week Before Wedding

## 3. Product Scope Confirmation

Confirm that the organizer understands what V1.0 includes.

Included:

- Guest list management
- QR/pass generation
- Offline staff check-in
- Manual guest search
- Check-in synchronization
- Guest photo/video upload
- Gallery/media moderation
- Basic dashboard stats

Not included in V1.0:

- RSVP automation
- Invitation builder
- Seating chart
- Vendor management
- Payment collection
- Full wedding planning tools
- Automatic social media posting
- AI photo/video curation

Checklist:

- [ ] Organizer understands V1.0 scope
- [ ] Organizer understands this is beta software
- [ ] Organizer agrees to provide feedback
- [ ] Organizer knows the product is free beta for feedback

## 9. Staff Training

- [ ] What to do if guest is already checked in

```text

If QR fails, search.
If guest is not found, send to manual desk.
Before leaving, sync.
```

```text
Scanner. Confirmer. Enregistrer. Continuer.

Avant de partir, synchroniser.
```


# Part 4 — During Check-In

## 15. Staff Operating Rules

Staff should:

- [ ] Scan QR first if available
- [ ] Search manually if QR fails
- [ ] Confirm guest name before check-in
- [ ] Check allowed guest count
- [ ] Continue if offline mode is active
- [ ] Send exceptions to manual desk
- [ ] Avoid spending too long on one problematic guest

Staff should not:

- [ ] Clear browser data
- [ ] Use private browsing
- [ ] Leave with pending sync
- [ ] Create new guests during Event Mode
- [ ] Panic when internet goes offline

# Part 6 — End of Event

## 19. Final Sync Closeout

This is mandatory.

Before staff leave:

- [ ] Device 1 pending sync = 0
- [ ] Device 2 pending sync = 0
- [ ] Device 3 pending sync = 0
- [ ] Device 4 pending sync = 0
- [ ] Any missing device identified
- [ ] Any failed sync documented
- [ ] Organizer confirms sync closeout

If any device has pending sync:

```text
Do not clear data.
Keep the device available.
Try Sync Now again when internet improves.
```

French instruction:

```text
N’effacez pas les données.
Gardez l’appareil disponible.
Réessayez la synchronisation quand Internet s’améliore.
```

## 24. Post-Beta Decision

After reviewing the wedding, decide:

```text
Proceed to next beta / Pause and fix issues
```

Proceed only if:

- [ ] No check-in data was lost
- [ ] Offline check-in worked
- [ ] Sync completed
- [ ] Staff could use the app
- [ ] Organizer understood the process
- [ ] Critical issues are documented or resolved

Pause if:

- [ ] Check-in data was lost
- [ ] Staff could not use the app
- [ ] Sync failed without recovery
- [ ] Public media privacy issue occurred
- [ ] Organizer lost trust
